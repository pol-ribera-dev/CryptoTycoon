# Gym Blockchain Tycoon

Gym Blockchain Tycoon is a Solidity-based idle game, where players earn tokens from NFTs. This NFT can be upgraded with time and tokens and can be traded for tokens with other players.

To optimize gas consumption, revert errors are represented by numeric codes. The description of each error can be found in the `errors.txt` file.

## How to use it

First of all, it is needed a deployed contract with all the parameters correct. These parameters define the game, things like how many tokens are needed to upgrade a NFT or the maximum level they can reach. My contract is already deployed with balanced params, so you can play on it.

Althought there are 5 contracts, all the functions must be call to [Main.sol](https://github.com/pol-ribera-dev/CryptoTycoon/blob/main/src/Main.sol).

### Start Playing

Calling start() mints the initial NFT set for the player and registers them as an active player. All NFT are lvl 1.

### Daily Rewards

Players can periodically claim ERC20 rewards generated by their NFTs.

The amount of tokens earned depends on:

- Number of NFTs owned (Except those that are currently upgrading)
- NFT levels
- NFT type

There are two types:
- Base NFTs - generate production.
- Multiplier NFTs - increase the total production.

### NFT Upgrades

Players can spend tokens to upgrade an NFT.

The process consists of:

1. Depositing the required amount of tokens.
2. Waiting until the upgrade timer finishes.
3. Claiming the completed upgrade.

If the player changes their mind, they can cancel the upgrade and receive a partial refund.

### Marketplace

The only way to trade NFT is using the one provided by the game.

Available actions include:

1. Listing NFTs for sale.
2. Purchasing listed NFTs.
3. Cancelling active listings.

When buying an NFT, the buyer pays both the listing price and the corresponding marketplace fee. This fee is to avoid people exploit a glich (get the reward of the same NFT two times same day).
